Atividade 006 - Banco de Dados da Agenda

  • Diagrama Entidade-Relacionamento

    O diagrama entidade-relacionamento do sistema apresenta duas entidades: usuários e seus respectivos eventos. Para a geração desde diagrama, foi utilizado o MySQL Workbench para a geração automatizada do esquema e das tabelas.

  • Dicionário de dados

    Com base no diagrama entidade-relacionamento mostrado anteriormente, abaixo consta o dicionário de dados descrevendo os atributos e seus tipos e tamanhos.

    Tabela: Usuário
    Atributo Classe Tipo Tamanho Descrição
    usuario_id Identificador INT 2147483647 Atributo que identifica um usuário
    usuario_nome Simples VARCHAR 100 Nome do usuário
    usuario_email Simples VARCHAR 100 E-mail do usuário
    usuario_status Simples INT 2147483647 Indica o status do usuário. Assume os valores: 0 (Deslogado), 1 (Logado), 2 (Deletado)
    Tabela: Evento
    Atributo Classe Tipo Tamanho Descrição
    evento_id Identificador INT 2147483647 Atributo que identifica um evento
    evento_titulo Simples VARCHAR 100 Título do evento
    evento_descricao Simples VARCHAR 200 Descrição do evento
    evento_data_hora Simples DATETIME - Indica o horário e a data que o evento irá ocorrer
    evento_status Simples INT 2147483647 Indica o status do evento. Assume os valores: 0 (Pendente), 1 (Cancelado)
    usuario_id Chave estrangeira INT 2147483647 Faz referência a chave-primária pertencente a um usuário
